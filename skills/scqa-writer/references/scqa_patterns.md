# SCQA 写作模式参考

本文档提供 SCQA 框架的深入指南，供需要详细了解时参考。

## 一、SCQA 变体模式

### 1. 标准式 (S-C-Q-A)
最经典的逻辑顺序，适用于大多数商业报告和技术文档。

**结构**：背景 → 冲突 → 问题 → 回答

**适用场景**：
- 项目提案
- 技术方案
- 商业分析报告

---

### 2. 开门见山式 (A-S-C)
先说结论，再补充背景和冲突。适合时间紧迫的决策者。

**结构**：回答 → 背景 → 冲突

**适用场景**：
- 高管汇报
- 紧急决策
- 电梯演讲

**示例**：
> **A**: 建议立即暂停项目 B，将资源转移到项目 A。
> **S**: 项目 A 和项目 B 同时启动，预计 Q2 交付。
> **C**: 但项目 B 的核心供应商宣布破产，交付风险极高。

---

### 3. 悬念式 (Q-S-C-A)
先抛出问题，制造悬念，再逐步揭示。适合演讲和教学。

**结构**：问题 → 背景 → 冲突 → 回答

**适用场景**：
- 演讲开场
- 培训课程
- 故事叙述

**示例**：
> **Q**: 为什么我们的用户留存率突然下降了 30%？
> **S**: 过去两年，我们的留存率稳定在 70% 以上。
> **C**: 上个月，竞争对手推出了免费版本，价格战开始。
> **A**: 我们需要在两周内推出差异化功能，而非降价。

---

## 二、Q 的验证技巧

Q 是 SCQA 的枢纽，决定了整个结构的逻辑性。

### 好的 Q 应该：
1. **由 C 自然引发**：读者看完 C 后，会自发产生这个疑问
2. **指向 A**：Q 的答案就是 A
3. **聚焦单一核心**：避免多个问题混杂

### Q 的三种类型

| 类型 | 典型问法 | 适用场景 |
|------|----------|----------|
| How | 该怎么办？ | 行动导向的方案 |
| What | 是什么？ | 定义或解释 |
| Why | 为什么？ | 原因分析 |

### 验证方法
1. 读完 S + C，自问：「我现在最想知道什么？」
2. 若受众自然产生的疑问与 Q 一致，则 Q 有效
3. 若 S + C 无法引导出 Q，需调整 S 或 C 的表述

---

## 三、金字塔原则在 A 中的应用

A 的结构应遵循金字塔原则：

```
        [核心结论]
       /    |    \
     [A1]  [A2]  [A3]
     /|\   /|\   /|\
   ...    ...   ...
```

### 原则

1. **结论先行**：先说核心观点，再展开论证
2. **归类分组**：子论点按逻辑归类（时间顺序、重要程度、因果关系）
3. **逻辑递进**：每层之间有清晰的推导关系
4. **MECE 原则**：子论点相互独立、完全穷尽

### 示例

**核心结论**：建议采用方案 A

- **A1 成本优势**：总成本比方案 B 低 30%
- **A2 时间优势**：实施周期缩短 2 个月
- **A3 风险可控**：技术栈与现有系统兼容

## 四、常见错误与修正

| 错误 | 问题描述 | 修正方法 |
|------|----------|----------|
| S 过长 | 背景铺垫太多，读者失去耐心 | 只保留与 C 直接相关的背景 |
| C 不够冲突 | 没有形成张力 | 强调变化、对比、影响 |
| Q 与 A 脱节 | A 没有直接回应 Q | 重新推导 Q 或调整 A |
| A 缺乏结构 | 论点混乱 | 应用金字塔原则重组 |

---

## 五、不同文体的 SCQA 应用

### 邮件
- SCA 控制在 3 段以内
- 主题行可直接用 A

### PPT
- 标题页用 A
- 第一页用 S + C
- 后续页展开 A1、A2、A3

### 技术文档
- 摘要部分用完整 SCQA
- 正文按 A 的金字塔结构展开
