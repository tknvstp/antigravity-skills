# 内部沟通专家

帮助撰写公司内部各类沟通文档，遵循标准化格式和风格。

## 适用场景

当用户要求撰写以下类型内容时使用：
- 3P 进度汇报（Progress, Plans, Problems）
- 公司通讯/新闻简报
- FAQ 问答
- 状态报告
- 领导层更新
- 项目更新
- 事故报告

## 工作流程

### 1. 识别沟通类型
根据用户请求判断属于哪种类型的内部沟通。

### 2. 读取对应指南

使用 `view_file` 工具读取示例文件。路径格式为 `{工作区根目录}/.agent/resources/internal-comms/examples/{文件名}`：

| 沟通类型 | 文件名 |
|---------|--------|
| 3P 进度汇报 | `3p-updates.md` |
| 公司通讯 | `company-newsletter.md` |
| FAQ 问答 | `faq-answers.md` |
| 其他通用 | `general-comms.md` |

**操作步骤**：
1. 确定工作区根目录（通常是用户当前项目的根路径）
2. 拼接完整路径：`{根目录}/.agent/resources/internal-comms/examples/3p-updates.md`
3. 调用 view_file 读取文件内容
4. 不要使用搜索，直接读取指定路径

### 3. 遵循具体指南
按照文件中的格式、语气和内容收集要求执行。

### 4. 模式切换（Mode）
- **信息收集阶段**：PLANNING 模式，了解团队、时间范围、受众
- **撰写阶段**：EXECUTION 模式，按模板生成内容
- **审核阶段**：VERIFICATION 模式，检查是否符合格式要求

## 快速参考：常用格式

### 3P 进度汇报格式
```
[emoji] [团队名称] (日期范围，通常一周)
Progress: [1-3句话，已完成的工作]
Plans: [1-3句话，下周计划]
Problems: [1-3句话，阻碍或问题]
```
- 30-60秒可读完
- 数据驱动，包含指标
- 语气简洁客观

### 公司通讯格式
```
:megaphone: 公司公告
- 公告1
- 公告2

:dart: 重点进展
- 领域1
    - 子项1
    - 子项2

:pillar: 领导层更新
- 更新1
- 更新2
```
- 20-25个要点
- 使用"我们"语气
- 包含大量链接

### FAQ 格式
```
- *问题*: [1句话问题]
- *回答*: [1-2句话回答，附链接]
```

## 通用原则

- 清晰简洁
- 使用主动语态
- 重要信息放在前面
- 包含相关链接和引用
- 匹配公司沟通风格
